# Backend-приложение для автоматизации закупок

## Описание проекта
Backend-часть интернет-магазина электроники с разделением ролей: покупатели, магазины и администраторы. Проект включает REST API, админ-панель Django с разными правами доступа в зависимости от типа пользователя, систему заказов, управление товарами и интеграцию с Celery для фоновых задач (отправка email).

## Технологический стек

- Backend: 
Django 5.2 + Django REST Framework

- База данных: 
PostgreSQL

- Очереди задач: 
Celery + Redis

- Контейнеризация: 
Docker + Docker Compose

- Email: 
SMTP (Mail.ru)

## Настройка окружения

В проекте уже есть файл-шаблон .env.example. Создайте рабочий файл .env на его основе.

## Запуск

- Собрать и запустить контейнеры:   
docker-compose up -d --build

- Выполнить миграции
docker-compose exec django python manage.py migrate:     

- Создать суперпользователя:   
docker-compose exec django python manage.py createsuperuser

## Пользователи сервиса

### Все пользователи могут:
- Авторизироваться;
- Регистрироваться;
- Восстанавливать пароль через API;
- Получать e-mail с токеном авторизации.

### Покупатель:
- Делает ежедневные закупки по каталогу, в котором представлены товары от нескольких поставщиков (добавление, изменение или удаление товаров из корзины);
- В одной корзине можно указать товары от разных поставщиков;
- Добавлять, изменять или удалять контакты (адреса доставки);
- Оформлять заказы из корзины (заказы при этом деляться на заказы для каждого магазина для возможности изменения статуса заказа магазином);
- Получает e-mail с информацией о создании заказа, изменении его статуса или количества товара в заказе.

### Поставщик:
- Через API информирует сервис об обновлении прайса;
- Может экспортировать прайс;
- Может включать и отключать приём заказов;
- Может менять статус заказа;
- Может менять количество товаров в заказе;
- Может получать список оформленных заказов (с товарами из его прайса).

## Админ-панель

### Cуперадмин (superuser):
- Полный доступ ко всем разделам;
- Полные права на создание/редактирование/удаление.
  
### Поставщик:
- Видит только свои товары;
- Видит только свои заказы;
- Может менять количество товаров в  своих заказах;
- Может удалять свои товары из заказов;
- Может менять статусы своих заказов.
  
### Покупатель:
- Видит все товары;
- Видит только свои заказы;
- Может добавлять, изменять или удалять свои контакты.
